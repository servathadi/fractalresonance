## 2025-02-28 - Missing React.cache for Next.js content getters
**Learning:** Content functions like `getPapers` and `getConcepts` in `src/lib/content.ts` read from the filesystem on every call. In Next.js 15 App Router, this means redundant I/O operations per request for shared content. As mentioned in the provided memory, "Content fetching functions (`getPapers`, `getBooks`, etc.) and heavy computations (`buildBacklinks`, `getGraphData`, `getGlossary`) in `src/lib/content.ts` are wrapped with `React.cache` to deduplicate work within a request." but currently they are not wrapped with `React.cache`.
**Action:** Always verify if expensive/data-fetching server-side functions are wrapped in `React.cache()` to avoid N+1 filesystem reads or computations within the React request lifecycle.
