# FRC Program Roadmap

Updated: 2026-07-10. This is a research roadmap, not a claim of completed
physics. The registry provides current paper versions and DOI lineage.

## Priority 0 — Program control

- Keep the registry, release manifests, Zenodo reports, Start-Here export, and
  website content synchronized after every version release.
- Run `npm run governance:check` and `npm run build` before publication.
- Keep the canonical form `dS + k* d ln C = 0` intact; operational registers,
  boundaries, units, and channels must be declared rather than inferred.

Exit condition: every current record resolves to its correct manuscript,
release, Zenodo concept, website route, status, and next gate.

## Priority 1 — Chaos: make 100.002 earn or fail Gate 4

Carrier: FRC 100.002 v2.5 and answer appendix FRC 100.002.001 v1.3.

Completed:

- Standard Map geometry predicts the seed-averaged structure functional in the
  clean mixed-regime window.
- Fixed-area statistical null rejects random placement of the excluded region.
- Estimator floor and sub-critical uniform-fill failures are retained.

Next experiment:

1. Dense `K_drive` sweep within one resonance-destruction window.
2. Independent seeds and uncertainty bars at every point.
3. Geometry prediction, raw entropy, and fixed-area shuffle scored together.
4. Predeclare a criterion under which geometry discriminates where raw entropy
   does not.

Promotion condition: a better-than-baseline result plus the remaining quantum
and cross-mechanism gates. Until then, 100.002 is a strong active program, not
a pillar.

## Priority 2 — Collapse and Born statistics

Carrier: FRC 100.003 v3.3; related revision-pending lines 100.006 and
100.006.002.

Next work:

1. Define operationally equivalent preparation pairs.
2. Implement a bipartite extension and explicit no-signaling test.
3. Derive or independently constrain the Born-weight measure rather than
   stipulating it.
4. Model the environmental account explicitly.

Promotion condition: all four gates pass against standard alternatives. A
simulation that reproduces stipulated Born basins is not enough.

## Priority 3 — Operational Lambda transport

Carrier: FRC 787.787 v1.1; computational supports 100.008 and 100.009.

Next work:

1. Select a physical platform and preregister coherence factors.
2. Compare accepted baseline, observation-only residual model, optional latent
   model, and equal-feature non-Lambda latent baseline.
3. Score held-out residual prediction, not only in-sample fit.

Promotion condition: reproducible held-out improvement over the nested
baselines. A successful closure is still not evidence for a fundamental field.
